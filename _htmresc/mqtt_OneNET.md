# mqtt_OneNET

Location: SIMCOM_SDK_SET/sc_demo/V1/src/mqtt_OneNET.c

## Overview

This file provides the MQTT credential configuration function for connecting to the China Mobile OneNET IoT platform. It generates the MQTT username, host address, client ID, and password (token-based) required for OneNET authentication. The password is computed using the SHA1-based token algorithm defined by OneNET's 2018-10-31 protocol version. This function is called by `OnenetDemo()` in `demo_mqtt.c`.

## Main Features

- Generates OneNET-specific MQTT connection parameters: username (product_ID), host address, client ID (device_name), and password (HMAC-SHA1 token)
- Constructs the OneNET connection URL (`tcp://183.230.40.96:1883`)
- Computes expiration time as current epoch + 1 year
- Uses `OTA_Authorization()` from `token.h` to generate the HMAC-SHA1 signed password
- Proper memory management with `sAPI_Malloc()`/`sAPI_Free()` for temporary buffers

## Usage

This function is not called directly by end users. It is invoked internally by `OnenetDemo()` in `demo_mqtt.c`:

1. The caller provides OneNET product ID, device name, and device secret.
2. The function fills output buffers with the MQTT connection parameters:
   - `cfg_username` = product_ID (up to 20 chars)
   - `cfg_host_addr` = "tcp://183.230.40.96:1883"
   - `cfg_client_id` = device_name (up to 20 chars)
   - `cfg_password` = HMAC-SHA1 token generated by `OTA_Authorization()`
3. The generated parameters are then used with `sAPI_MqttAccq()` and `sAPI_MqttConnect()`.

### Token Generation Algorithm

The password is computed as follows:
1. Build a resource string: `products/<product_ID>/devices/<device_name>`
2. Calculate expiration time: `current_epoch + 365*24*60*60` (1 year from now)
3. Call `OTA_Authorization(version="2018-10-31", res, et, device_secret, password_buf, buf_len)`
4. The result is an HMAC-SHA1 signed token string used as the MQTT password

## Implementation Notes

- The host address is hardcoded to `tcp://183.230.40.96:1883`, which is OneNET's standard MQTT endpoint.
- The username is simply the product ID, and the client ID is the device name -- these are OneNET's standard identification scheme.
- The method is SHA1 as specified by OneNET protocol version 2018-10-31.
- A section with `#if 0` shows an alternative approach using `sAPI_GetSysLocalTime()` for time calculation, but the active code uses `sAPI_Time()` for epoch time.
- Temporary buffers (`version`, `ten_res`, `method`, `password`) are allocated dynamically and freed after use.
- The function signature `Cfg_mqtt_service` is declared as `extern` in `demo_mqtt.c` for cross-file linkage.

## Best Practices

- Ensure the module's system time is synchronized (via NTP or HTP) before calling this function, as the token expiration time depends on accurate system time.
- The generated password/token has a 1-year validity period by default. Adjust the expiration offset for your security requirements.
- Store device secrets securely; they should never be hardcoded in production firmware.
- The host address `183.230.40.96:1883` is the non-SSL endpoint. For production deployments, consider using the SSL/TLS endpoint with `sAPI_MqttSslCfg()`.
